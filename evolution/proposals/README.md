# Proposals

Proposals are the governed bridge between observations and library changes.

A proposal should exist only when there is enough evidence to suggest that the library itself — not only the local project — may benefit from refinement.

## Directories

- `approved/` — proposals accepted for future library writeback
- `rejected/` — proposals reviewed and declined
- `deferred/` — proposals kept for later because the evidence is still too thin or the timing is wrong

A proposal is not the same thing as a merged skill change.
The actual write step still happens through a normal pull request against the canonical skill files.
