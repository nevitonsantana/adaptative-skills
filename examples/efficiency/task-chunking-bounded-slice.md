# Task Chunking — Bounded Slice Example

Use `task-chunking` when a request arrives as one oversized unit but the next safe move should be much smaller.

## Starting request
Improve the whole crisis-operations dashboard with copy fixes, tracking, and review-flow changes.

## Bounded slice
- focus on one summary surface
- keep the change reviewable
- keep proof local to that surface
- define what must wait

## Why this is a good chunk
- it is visible enough to teach something
- it does not silently import the whole dashboard initiative
- it gives the next round a clear trigger instead of a vague continuation
